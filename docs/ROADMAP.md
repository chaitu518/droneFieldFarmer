# 4-Week MVP Roadmap (Quick Booking First)

## Week 1
- Scaffold Spring Boot backend and Vite + React frontend
- OTP auth (request/verify)
- quick booking UI button + API wiring

## Week 2
- geolocation capture + manual location fallback
- single active booking rule (backend + frontend handling)
- booking status view (`/bookings/me/active`)

## Week 3
- admin assignment flow (manual scheduling ops)
- job completion with verified area
- final pricing calculation

## Week 4
- payment integration
- notifications
- bug fixing + end-to-end QA
- deployment baseline

## Launch Checklist
- OTP rate limiting tested
- one-active-booking rule tested (`409` path included)
- location permission denied fallback tested
- payment reconciliation tested
- basic monitoring and error logging enabled
